# Activation And Boundaries

## When To Activate

Activate this skill only when the user explicitly wants cross-tech-stack expansion, or when the repository is clearly one of:

- mobile app project
- H5 / web frontend project
- Python web / worker / ETL / script project
- mixed-stack repository spanning client + frontend + backend + task + callback + bridge

## When Not To Activate

Do not activate this skill for ordinary backend-only microservice analysis.
In those cases, keep using the backend-service-spec-skill alone.

## Relationship With The Existing Skill

This skill does not replace the backend-service-spec-skill.
It extends it in stack types where backend-service terminology is too narrow.

## Key Rule

Preserve all evidence and quality rules from the backend-service-spec-skill.
This skill changes adaptation and vocabulary, not fact standards.
